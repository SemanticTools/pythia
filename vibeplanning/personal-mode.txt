Personal mode means.
We are admin. We can do anything.
No login. (keep mind for future)
Sleek hacker-ish gui.

1. tabs.  main tab is is discussion tab.
2. second tab is upload tab.

in the main tab, we can unhide a section called quick-memory.
Here you can 
1. add notes (will be appended to username-notes.txt)
2. edit quick-memory (allow to see whole file and make edits, but gui will ask twice "are you sure to save")

We will also make sources more granular.
Not it is just <filename>
But we will also allow <filename>::<snippetid>

A snippet is indicated like this in a text file

%% <snippet name (only alfanum, rest ignore)>

second tab will have upload tab.
Here we can upload file
List the files.
Rename a file.
Disable a file. (filename convention, prefix = "_d_" )
Enable a file (if disabled)

Note "%%" string and "_d_" string will be configurable in toml file again.
